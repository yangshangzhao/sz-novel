## 📖 项目简介

**SZ-Novel** 是一个基于 Claude AI 的深度学习型长篇小说创作专家系统。采用"先学后写"的创新工作范式，支持多卷多章的长篇小说创作，自动确保故事连贯性和质量标准。

### 🎯 核心特性

- **智能学习系统**：从参考资料中深度学习写作风格、人设逻辑和世界观设定
- **纯中文创作**：零容忍非中文内容，确保语言纯度
- **质量保障体系**：6维质量检查 + 实时纠错机制
- **连贯性保证**：防止章节缺失、设定漂移、人设突变
- **去AI化处理**：5项反检测策略，避免被AI检测工具识别
- **自动化持久化**：智能文件管理，按卷章自动保存

## �📋 工作流程

### 🔄 新建创作模式

1. **环境扫描** → 验证参考资料目录
2. **深度学习** → 分析参考资料风格
3. **内容创作** → 生成高质量章节
4. **自动化持久化** → 智能保存文件

### 🔄 继续创作模式

1. **上下文检索** → 扫描项目目录读取上次内容
2. **上下文学习** → 提取人物、设定、伏笔信息
3. **连贯性保障** → 确保新章节与前文无缝衔接
4. **异常修复** → 自动补齐缺失章节

## 📊 质量标准

### 6维质量检查体系

#### 📖 维度1：故事清晰度与节奏
- ✅ 主线清晰，故事推进明确
- ✅ 节奏张弛有度，高潮缓和交替
- ✅ 悬念与伏笔合理布局，禁止"只埋不收"

#### 👥 维度2：人设统一性与成长
- ✅ 人物言行符合性格、出身、动机
- ✅ 人物关系合理且动态演变
- ✅ 主角成长弧线完整可信

#### 🌍 维度3：内部自洽性
- ✅ 世界观规则严格遵守
- ✅ 禁止前后矛盾（如"真气耗尽会死"却安然无恙）
- ✅ 特殊机制（如魔法、穿越、科技）逻辑一致

#### 🧠 维度4：角色智商与冲突合理性
- ✅ 反派足够强大聪明，有实质威胁
- ✅ 主角胜利基于努力、智慧而非降智或天降神兵
- ✅ 所有冲突都有合理因果

#### 📝 维度5：基本信息一致性
- ✅ 人物信息统一（眼睛颜色、家庭背景等）
- ✅ 时间线准确（昨天周一，今天不能周三）
- ✅ 地点场景描写一致

#### 🔗 维度6：因果关系合理性
- ✅ 重大转折有充分内在原因和外部推动
- ✅ 人物决定基于当前信息和性格
- ✅ 禁止无原因的立场转变

### 🛡️ 去AI化策略

- ✅ **无效细节**：插入无关琐碎信息
- ✅ **逻辑跳跃**：场景转换偶尔不顺畅
- ✅ **情感毛边**：情感表达不完全、不确定
- ✅ **故意犯规**：打破完美排比，词汇搭配别扭
- ✅ **个人怪癖**：口头禅、非常规审美视角

## ⚙️ 技术规格

### 创作参数

| 参数     | 规格             |
| -------- | ---------------- |
| 单章字数 | 2000～4000字     |
| 全书规模 | 10～30章/卷      |
| 语言要求 | 纯中文（零容忍） |
| 保存格式 | UTF-8 文本文件   |

### 质量控制

- **实时扫描**：每50字自动检查
- **自动纠错**：发现问题立即修正
- **三重验证**：生成→补齐→保存前验证
- **零容忍政策**：任何质量问题都必须解决才能保存

## 🔧 高级功能

### 章节序号管理

系统采用**绝对零容忍**的序号管理机制：

```
已有章号：1, 2, 3, 5, 6, 8, 9
↓
缺失检测：发现缺失第4、7章
↓
自动补齐：先生成第4章，再生成第7章
↓
质量验证：每个补齐章节都执行完整检查
↓
连贯确认：全部补齐后重新验证序号完整
```

### 上下文接续

**继续创作模式**自动执行：
- 📖 扫描项目目录读取上次内容
- 🧠 提取关键信息（人物、设定、伏笔）
- 🔗 确保新章节无缝衔接
- ⚡ 异常修复（补齐缺失章节）

## ⚠️ 注意事项

### 禁止事项

- ❌ **非中文内容**：任何英文、其他语言混入
- ❌ **章节缺失**：不能跳过任何规划的章节
- ❌ **设定漂移**：人物、世界观前后矛盾
- ❌ **AI特征明显**：过度完美、排比、对偶
- ❌ **元叙事评论**：作者声音混入故事
- ❌ **伏笔遗忘**：只埋不收的悬念

### 最佳实践

- 📚 **准备参考资料**：放入 `references/` 目录供学习
- 🔄 **分批创作**：建议一次完成1-3章确保连贯
- ✅ **质量确认**：每次保存前仔细检查质量报告
- 📝 **纯中文环境**：确保所有输入输出都是纯中文

## 📈 质量保证

### 验证流程

1. **实时检查**：创作过程中每50字扫描
2. **章节验证**：单章完成后质量检查
3. **连贯验证**：多章时检查序号和设定一致性
4. **保存验证**：最终确认无任何问题

### 错误处理

- **自动纠错**：发现问题立即停止并修正
- **补齐机制**：缺失章节自动生成补齐
- **验证失败**：不符合标准无法保存，必须修复
