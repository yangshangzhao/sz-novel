## 📖 项目简介

**SZ-Novel** 是一个面向长篇小说创作的深度学习写作专家，采用“先学后写”范式。通过分析参考资料提取人物逻辑、世界观规则和文风特征，辅助生成连贯、高质量、纯中文的章节内容。

## 🎯 核心功能

- 智能学习：从参考资料中提取写作节奏、人设和设定逻辑
- 纯中文创作：零容忍非中文内容，保证语言纯度
- 多维质量检查：实时扫描与阶段性核验相结合
- 章节连贯保障：防止跳号、设定漂移、人物矛盾
- 去 AI 味：依据 `DEAIFY.md` 规则规避 AI 特征
- 自动持久化：智能文件保存与章节管理

## 🚀 使用方式

### 新建创作
直接开始写作，系统进入“新建创作模式”。

### 继续创作
输入“继续创作”，系统进入“继续创作模式”，自动读取上一章内容并继续写作。

## 🔧 关键流程

### 0. 启动检查与模式选择

- 指令包含“继续创作”时进入继续创作模式
- 否则进入新建创作模式

### 继续创作模式

- 扫描项目根目录最近修改的 `.txt` 文件
- 提取最后章节号、结尾内容、人物、伏笔和时间线
- 如无异常直接继续写作；若发现缺失、错序或污染，先修复再继续
- 保证新章节与前文自然衔接

### 新建创作模式

1. 环境扫描：检查 `references/` 目录并评估参考资料
2. 深度学习：分析参考资料的文风、节奏、人设与世界观
3. 内容创作：按标准生成单章内容
4. 自动保存：保存前进行最终校验

## ✅ 内容创作规则

- 章节长度：2000～4000 字
- 事前确认章节号序列，禁止跳号
- 实时检测：每 50 字扫描一次非中文和逻辑问题
- 发现问题立即修正后继续写作
- 不在“看似完整”处停止，必须满足章完整性
- 若发现缺失章节，先补齐再保存

## 📊 质量标准

### 6 维质量检查

1. 故事清晰度与节奏
2. 人设一致性与成长
3. 内部自洽性
4. 智商与冲突合理性
5. 基础信息一致性
6. 因果关系合理性

### 去 AI 味策略

- 插入无效细节和环境琐碎
- 制造逻辑跳跃与情绪毛边
- 避免工整排比、情绪标签动词、过度完美结论
- 加入人物口头禅和不完美表达
- 参考 `DEAIFY.md` 进行调整

- 文件名与标题须为纯中文
- 自动管理章节序号，确保无缺失

## ⚠️ 重要事项

### 禁止项

- 任何非中文内容
- 章节跳号或缺失
- 设定漂移与人物矛盾
- 明显 AI 写作特征
- 元叙事评论或读者视角表达
- 只埋伏笔不回收

### 推荐实践

- 先准备参考资料再写作
- 推荐一次写 1～3 章，保证连贯性
- 定期回顾前文，修复错别字与设定问题
- 每章完成后立即进行完整质量检查

## 📈 验证流程

1. 实时检查（每 50 字）
2. 章节完成后质量验证
3. 连续章节一致性检查
4. 保存前最终确认
